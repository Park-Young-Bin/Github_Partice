4. 깃허브로 백업하기

- 지역 저장소(local repository)
자신의 컴퓨터에서 작업한 뒤 그 컴퓨터 안에 커밋을 저장해서 보관하는 방법이다.
지역 저장소를 삭제한다면 작업했던 내용이 사라지기 때문에 지역 저장소안에 저장하는 것은 안전하지 않다.

- 원격 저장소(Remote repository)
지역 저장소와 원격 저장소를 연결해서 버전 관리하는 파일들을 쉽게 백업할 수 있다.
원격 저장소는 컴퓨터나 서버에 만든 저장소를 의미한다. 특히 원격 저장소는 지역 저장소와 연결되어 있기 때문에 [ 백업 ] 과 [ 협업 ] 이라는 중요한 역할을 담당한다.

- 깃허브(GitHub)
깃 사용을 위한 원격 저장소를 제공하는 서비스이다.
사용자를 위해 저장소를 제공하며 유료와 무료 서비스가 있다.
깃허브는 많은 개발자들이 만든 여러 오픈 소스가 올라와 있으며 무료 서비스만으로도 용량 제한 없이 사용할 수 있는 저장소이다.

★ 깃허브로 할 수 있는 일 ★
- 원격 저장소에서 깃을 사용할 수 있습니다.
- 지역 저장소를 백업할 수 있습니다.
- 협업 프로젝트에 사용할 수 있습니다.
- 자신의 개발 이력을 남길 수 있습니다.
- 다른 사람의 소스를 살펴볼 수 있고, 오픈 소스에 참여할 수도 있습니다.

[명령어]
- git remote add origin *복사한 주소 붙여넣기*:  원격 저장소(remote)에 origin을 추가(add)한다.
→ 'remote'는 '원격 저장소'를 뜻한다.
→ 'origin'은 깃허브 저장 주소(http://github.com/...)를 가리킨다.
- git remote -v: 지역 저장소가 원격 저장소에 연결되어 있는지 확인할 수 있다.
- git push -u origin master: 지역 저장소의 브랜치(origin)를 원격 저장소의 master 브랜치로 푸시한다.(원격 저장소에 파일 올리기)
([-u] 옵션은 지역 저장소의 브랜치를 원격 저장소의 master 브랜치에 연결하기 위한 것으로 처음에 한 번만 사용 된다.)
- git pull: 원격 저장소에서 파일을 내려받을 수 있다.(원격 저장소에서 파일 내려받기)
- git pull origin master: 원격 저장소(origin)의 내용을 master 브랜치로 가져온다.
