5. 깃허브로 협업하기

git_home: 개인 pc/노트북 저장소
got_office: 회사 컴퓨터 저장소
→ 하나의 깃허브 계정으로 둘 이상의 컴퓨터에서 원격 저장소를 공유해 버전을 관리한다.

※ 원격 저장소를 기존에 연결된 지역 저장소 외에 다른 지역 저장소에서 사용하려면 원격 저장소에 담긴 내용 전체를 지역 저장소로 가져와야 한다.
- 클론(clone) or 클로닝(cloning): 원격 저장소를 지역 저장소로 똑같이 가져오는 것을 말한다.
- git clone: 원격 저장소를 복제할 수 있다.
- git clone (복사한 주소) (복제할 디렉터리 이름): 원격 저장소를 복제하면 자동으로 지역 저장소와 원격 저장소가 연결된다.
- git remote -v: 연결되었는지 확인할 수 있다. 
ex. git_home(개인)과 git_office(회사)를 clone한 후에 각 디렉터리로 이동 후(cd 디렉터리 명), git log를 실행하면 똑같은 커밋이 저장되어 있을 것이다.

[과정]
1. 개인 pc에서 git push 명령을 사용해 커밋을 깃허브(원격 저장소)에 올린다.
2. 회사에서 git pull 명령을 이용하여 깃허브(원격 저장소)에 새로 올라온 커밋을 가져온다.
3. 회사에서 내용을 수정하고 푸시(git push)하여 깃허브(원격 저장소)에 다시 올린다.
4. 개인 pc에서 git pull 명령으로 깃허브(원격 저장소)에 있는 최신 커밋을 가져와 작업한다.

지역 저장소의 master 브랜치처럼 원격 저장소도 만들 때 기본으로 master 브랜치가 생성된다.
여기에서 HEAD -> master는 이 커밋이 [ 지역 저장소의 최종 커밋 ] 이라는 뜻이고 origin/master는 [ 원격 저장소의 최종 커밋 ] 이라는 뜻이다.

※ 원격 저장소에서 협업을 할 때는 원격 저장소의 최신 커밋 정보를 가젼온 다음, 지역 저장소의 커밋을 올려야 한다.
※ 깃허브로 협업할 때는 다른 작업자의 변경 내용을 바로 반영하기 위해 항상 (pull)부터 한 다음 자신의 작업을 진행하는 것이 좋다.
​
- git fetch: 원격 브랜치 정보를 가져올 수 있다.
- cd ~/git_office: cd ~명령과 cd git_office 명령을 한꺼번에 처리한다.
- git checkout -b fixed: fixed 브랜치를 만드는 것과 동시에 체크아웃한다.
- git push origin f: 원격 저장소에 f 브랜츠의 커밋을 올린다.
